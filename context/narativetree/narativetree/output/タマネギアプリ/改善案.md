# 概要
`output/タマネギアプリ/改善案.md`は既存ドキュメントを最新コンセプトと統合するための改善指針をまとめた設計ノート。コンセプトの再言語化、機能群の詳細化、LLM・安全設計・KPIの明示といった項目別に、どこへ何を追記すべきかのロードマップを提示する。

# 目的
- tamanegicontext/TAMANEGIで蓄積された知見（スコアリングやUXモード、研究根拠）を、最新の`output/タマネギアプリ`配下へ反映するための着手順を定義する。
- 各ディレクトリの不足点を可視化し、優先して埋めるべきドキュメント箇所を合意形成する。

# 最適な実装方法
1. 改善案に沿って既存ファイルへ追記 or 新規ファイルを作成し、都度ナラティブツリーを更新する。
2. コア体験（育てる・剥がす・フォロワー）から着手し、スコアリングと安全要件を同時に記述して抜け漏れを防ぐ。
3. 研究エビデンスやKPIなど、後回しになりがちな情報は専用ファイルを追加し、関連セクションから参照リンクを貼る。

# 最適と思われる理由
- 過去ドキュメントに存在する詳細（深さスコア、Affinity閾値、トラスト構築UX）が現行資料から欠落しており、開発判断に必要な文脈が不足しているため。
- 改善案を章立てで整理することで、複数人が並行してドキュメント整備を進めても認識齟齬が生じにくい。

# 入出力
- 入力: 現状の`output/タマネギアプリ`ドキュメント、tamanegicontext配下の過去資料、改善案ドキュメント。
- 出力: 具体的な追記タスク、追加すべきファイル案、優先順位付きの次アクション。

# 実装の評価
- 改善案に記載した各セクションの更新が完了し、ナラティブツリーとログに反映されているかで評価する。
- コンセプト・機能・安全性・KPIの要件が互いに矛盾なく説明され、MVPのスコープが明文化されているかを確認指標とする。
