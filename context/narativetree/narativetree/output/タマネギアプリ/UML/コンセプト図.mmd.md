# 概要
`output/タマネギアプリ/UML/コンセプト図.mmd`は、玉ねぎアプリの根幹となる動機・体験原則・システム構成をMermaidのUMLクラス図で整理したもの。

# 目的
- コンセプトドキュメントで言語化された価値提供要素を俯瞰し、どのコンポーネントがユーザー課題に対処するのかを可視化する。

# 最適な実装方法
1. `classDiagram`を用いてアプリ全体（TamaNegiApp）と関連要素（動機、体験ピラー、システム構成）を表現。
2. サブコンポーネント（EpisodeEditor、OnionLayerSharingなど）はコンポジションで結び、役割を表示。

# 最適と思われる理由
- UMLクラス図は要素間の責務と依存関係を示すのに適しており、コンセプト段階の整合性チェックに活用できる。

# 入出力
- 入力: コンセプト文書の要約（背景課題、コア体験、LLM活用、安全設計）。
- 出力: Mermaidコードによる可視化。

# 実装の評価
- 図からコンセプトの3レイヤー（動機・原則・システム）が読み取れ、レビュー時に依存関係を説明しやすいこと。
