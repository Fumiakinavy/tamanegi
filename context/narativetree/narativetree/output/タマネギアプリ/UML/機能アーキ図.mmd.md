# 概要
`output/タマネギアプリ/UML/機能アーキ図.mmd`は、玉ねぎアプリの主要機能モジュールとデータレイヤの関係を示すUMLクラス図。

# 目的
- エピソードエディタ、マッチング、共有、安全、成長機能がどのように連携し、データレイヤへアクセスするかを俯瞰する。

# 最適な実装方法
1. 各機能をクラスとして定義し、役割をメンバで簡潔に記述。
2. モジュール間の依存関係を関連線で表現し、安全モジュールが横断的に関与する点を明示する。

# 最適と思われる理由
- クラス図により責務境界とデータアクセスパターンが明瞭になり、設計レビューでの議論が容易。

# 入出力
- 入力: `output/タマネギアプリ/機能/`配下の仕様メモ。
- 出力: Mermaidコードによる機能アーキテクチャ図。

# 実装の評価
- 主要機能が漏れなく図に含まれ、依存関係が仕様意図と一致しているかをチームで確認できること。
