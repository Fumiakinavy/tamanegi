# 概要
`output/タマネギアプリ/UX/リッチUML実装ガイド.md`は、Next.js + React Flow + Tailwindなどのフロントエンド技術を使ってUML図をインタラクティブに提供するための実装ガイドラインをまとめた文書。

# 目的
- UXドキュメントとして、リッチなフロントエンドスタックに基づくUML表示の構成要素や手順を共有する。
- Mermaidによる静的図とReactコンポーネントを同期させ、ドキュメントとアプリ実装の乖離を防ぐ。

# 最適な実装方法
1. Next.jsプロジェクトにReact Flow、Tailwind、shadcn/uiを導入し、UXページに`UserJourneyFlow`を組み込む。
2. Mermaid CLIやJSONスキーマでノード定義を共通化し、Markdown図とReact Flowを同一ソースから生成。
3. StorybookやPlaywrightで操作性とアクセシビリティを検証する。

# 最適と思われる理由
- React Flowが提供するインタラクションとTailwindのテーマ管理により、設計レビューやユーザーテストで使いやすい環境を実現できる。
- ガイドを通じて実装チームとデザインチームの共通認識を持たせ、将来の拡張（フィルタリング、ハイライト）にも対応しやすい。

# 入出力
- 入力: 技術スタックの選定情報、ディレクトリ構成案、実装ステップ。
- 出力: インタラクティブなUMLダイアグラムを持つNext.jsページの構築手順と運用留意点。

# 実装の評価
- ガイド通りに進めることでReact FlowベースのUMLが動作し、アクセシビリティやテスト方針もカバーできているかを確認。
- MermaidとReact Flowの両方で同一ノード定義が再利用できる構成が実現しているかを指標とする。
