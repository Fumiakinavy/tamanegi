# 概要
`output/タマネギアプリ/UX/ユーザー体験フローUML.md`は、玉ねぎアプリにおけるユーザー体験をUMLアクティビティ図で可視化した資料。ホーム画面から「育てる」「剥がす」「フォロワー」へ分岐する主要導線と、その後のマッチングやリフレクションまでの流れをまとめている。

# 目的
- 既存コンセプトを変更せず、体験シーケンスの全体像を共有することで、設計・実装チーム間の認識合わせを容易にする。
- エピソード作成、相互開示、関係管理の各フローで必要な状態遷移を洗い出し、後続のバックエンド／マッチング設計の前提とする。

# 最適な実装方法
1. Mermaidの`stateDiagram-v2`を用いてアクティビティ図を記述し、主要状態と遷移を明示する。
2. ホームからの3導線をサブステートとして表現し、各導線の完了後はホームへ戻るループとなるよう設計する。

# 最適と思われる理由
- UMLでフローを可視化することで、既に確定しているコンセプトを崩さずにユーザー行動を定義できる。
- Mermaidを使うことでバージョン管理下でのレビューと変更追跡が容易。

# 入出力
- 入力: 既存コンセプトドキュメントのコア体験定義。
- 出力: ユーザー体験の主要状態と遷移が一覧化されたUML図。

# 実装の評価
- コンセプトの変更を伴わず、ホーム→導線→戻りの循環構造が明確である。
- 今後の機能詳細化やマッチングフロー設計と矛盾がないかをレビューし、必要に応じて追加ステートを検討する。
